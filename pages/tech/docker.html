Docker

Docker permite empaquetar aplicaciones junto con todo lo que necesitan para funcionar, garantizando que corran igual en la laptop de un desarrollador que en producción.

¿Qué es Docker?

Docker es una plataforma de contenedores que permite empaquetar una aplicación junto con sus dependencias —librerías, configuración, sistema de archivos— en una unidad portable llamada contenedor.

Su propósito es eliminar el clásico problema de "en mi máquina funciona": un contenedor Docker corre exactamente igual sin importar dónde se ejecute, desde la laptop de un desarrollador hasta un servidor en la nube.

Lanzado en 2013, Docker popularizó la tecnología de contenedores —que ya existía en Linux— haciéndola accesible y fácil de usar, y se convirtió rápidamente en el estándar de facto de la industria.

Hoy se usa en prácticamente todo el ciclo de vida del software: desarrollo local, testing, integración continua, y como unidad básica de despliegue en plataformas de orquestación como Kubernetes.

¿Cómo funciona?

Imágenes

Una imagen Docker es una plantilla de solo lectura con todo lo necesario para ejecutar una aplicación: código, runtime, librerías y configuración.

Contenedores

Un contenedor es una instancia en ejecución de una imagen, aislada del resto del sistema pero mucho más liviana que una máquina virtual completa.

Dockerfile

Un archivo de texto que describe paso a paso cómo construir una imagen, haciendo que el proceso sea reproducible y versionable como cualquier otro código.

Docker Compose

Una herramienta para definir y ejecutar aplicaciones con múltiples contenedores (API, base de datos, caché) con un solo comando.

Registro de imágenes

Repositorios como Docker Hub donde se almacenan y comparten imágenes, de forma similar a como Git gestiona el código fuente.

Casos de Uso

Equipo de desarrollo con entornos inconsistentes

Problema: "en mi máquina funciona": errores que solo aparecían en producción.
Solución: containerización de la aplicación y sus dependencias con Docker.
Resultado: eliminación casi total de errores relacionados con el entorno.

Empresa con despliegues manuales y lentos

Problema: poner en producción una nueva versión tomaba horas y requería pasos manuales.
Solución: pipeline de build y despliegue basado en imágenes Docker.
Resultado: despliegues reducidos de horas a minutos.

Startup con infraestructura sobredimensionada

Problema: cada servicio corría en su propia máquina virtual, con recursos subutilizados.
Solución: migración a contenedores Docker sobre infraestructura compartida.
Resultado: 35% menos costos de infraestructura.

Buenas Prácticas

Mantené las imágenes livianas

Menos capas y dependencias innecesarias significan builds y despliegues más rápidos.

No corras como root

Ejecutar procesos con un usuario sin privilegios reduce el impacto de una vulnerabilidad.

Versioná las imágenes con tags claros

Evitá depender de "latest" en producción para mantener despliegues predecibles.

Separá configuración sensible

Variables de entorno y secretos nunca deben quedar embebidos en la imagen.

Escaneá imágenes en busca de vulnerabilidades

Herramientas de escaneo detectan librerías desactualizadas dentro de tus imágenes.

Limpiá recursos no utilizados

Imágenes y contenedores huérfanos consumen espacio y generan confusión innecesaria.
