React

React es la librería de JavaScript más utilizada para construir interfaces de usuario interactivas, y la base del frontend de gran parte de las plataformas web que desarrolla PATERSAY.

¿Qué es React?

React es una librería de JavaScript de código abierto, creada por Facebook (hoy Meta) y liberada en 2013, diseñada para construir interfaces de usuario mediante componentes reutilizables.

Su propósito es simplificar la construcción de interfaces complejas y dinámicas, dividiendo la pantalla en piezas independientes que se combinan y actualizan de forma eficiente cuando cambian los datos.

Nació en 2013 dentro de Facebook para resolver problemas de rendimiento en interfaces con mucha interacción; con el tiempo introdujo los Hooks (2019), que simplificaron la gestión de estado y lógica reutilizable sin depender de clases.

Hoy es la base de aplicaciones de una sola página (SPA), portales corporativos, dashboards y, mediante React Native, también de aplicaciones móviles nativas.

¿Cómo funciona?

Componentes

La interfaz se divide en piezas reutilizables que reciben datos (props) y devuelven la interfaz que deben mostrar.

DOM virtual

React mantiene una representación en memoria de la interfaz y actualiza solo las partes que realmente cambiaron, en lugar de redibujar toda la pantalla.

Hooks y estado

Funciones como useState y useEffect permiten manejar el estado y los efectos secundarios de un componente sin necesidad de clases.

Renderizado declarativo

En lugar de indicar paso a paso cómo modificar la pantalla, se describe cómo debe verse la interfaz para cada estado posible.

Ecosistema de frameworks

Herramientas como Next.js extienden React con renderizado en servidor, rutas y optimizaciones listas para producción.

Buenas Prácticas

Mantené componentes pequeños

Componentes enfocados en una sola responsabilidad son más fáciles de testear y reutilizar.

Elegí bien dónde vive el estado

No todo necesita un store global; el estado local es suficiente en la mayoría de los casos.

Testeá los componentes

Pruebas automatizadas evitan regresiones visuales y funcionales.

Evitá renders innecesarios

Memoización y buenas prácticas de props mejoran el rendimiento percibido.

Diseñá con accesibilidad

Contraste, navegación por teclado y semántica HTML desde el inicio, no como parche.

Usá code splitting

Cargá solo el código necesario para cada pantalla, reduciendo el tiempo de carga inicial.
